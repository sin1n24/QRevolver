# QRevolver — The Quick-Scan Showdown, Reloaded

[QRGunman](https://github.com/sin1n24/QRGunman)（M5Stack版・早撃ちQRゲーム）の **Web移植・改良版**です。
M5Stackの実機が不要になり、**PCや別のスマホでWebサイトを開くだけ**で「早撃ち台」になります。
画面に表示されたQRコードを、挑戦者がスマホでどれだけ速くスキャンできるかを競います。

```
┌─────────────┐   ①QR出現      ┌──────────────┐
│  ディスプレイ  │  ───────────▶  │  挑戦者のスマホ  │
│ (PC/別スマホ) │   ②スキャンで    │   (カメラ)    │
│  = 早撃ち台   │  ◀───────────  │              │
└─────────────┘   到達=タイム確定  └──────────────┘
```

## 計測の仕組み（ここが移植のキモ）

元のM5Stackは「**自分自身がWebサーバー**」だったため、
QRを表示した時刻 と スマホがアクセスしてきた時刻 を **同じ時計（millis）** で測れました。
だから端末間の時計ズレに左右されず、正確な早撃ちタイムが出せます。

このQRevolverでも **Node.jsサーバーが同じ役割**を担います。

1. ディスプレイで「抜き始め」→ サーバーが3〜5秒のランダム待機
2. 待機後 **fire**：サーバー時刻 `fireTime` を記録し、QR（毎回違う使い捨てURL）を表示
3. スマホがQRをスキャン→ブラウザが `/hit` にアクセスした瞬間、サーバーが到達時刻を記録
4. `到達時刻 − fireTime` を**同一サーバー時計で計算** → これが早撃ちタイム

QRはラウンド毎にURLが変わるため、**事前スキャン・先読みのチートを防止**します。

## 公開して誰でも遊べるようにする（自分のPC不要・推奨）

無料クラウド **Render** にデプロイすると、常時稼働の公開URLが手に入り、
**自分のPCを起動していなくても**誰でもアクセスして遊べます。
QRに埋め込む公開URLは Render のドメインから自動で決まる（`RENDER_EXTERNAL_URL`）ので設定不要です。

### 手順
1. このプロジェクトを GitHub に push（リポジトリは公開でOK）
2. [Render](https://render.com) にサインアップ → **New +** → **Blueprint**
3. 対象の GitHub リポジトリを選ぶ（同梱の `render.yaml` を自動検出）
4. デプロイ完了後に表示される `https://qrevolver-xxxx.onrender.com` を
   **ディスプレイ端末（PC等）で開く** → そのまま早撃ち台になる

> Blueprintを使わない場合：**New + → Web Service** で同リポジトリを選び、
> Build `npm install` / Start `npm start` を指定するだけでも可。

### 無料プランの注意
- 一定時間アクセスが無いとスリープし、**復帰時の初回アクセスが数十秒遅い**ことがあります。
- Render自体のディスクは揮発性なので、ランキングの**恒久保存は下記のGAS連携**を使ってください
  （GAS未設定時はローカル`records.json`に保存され、再デプロイで消えます）。

## ランキングの永続化（GAS＋スプレッドシート・無料）

ランキングは **月ごとにバケット化**され、**毎月自動でリセット**されます（例：7月になると「7月のランキング」が
空から始まり、6月分は履歴としてスプレッドシートに残ります）。月の境界は日本時間(JST)基準です。

恒久保存には、Googleスプレッドシートを無料データベースにする **GAS Webアプリ**を使います
（追加アカウント不要・Renderのディスクに依存しない）。

### GAS側の設定
1. 新しいGoogleスプレッドシートを作成
2. 拡張機能 → **Apps Script** を開き、同梱の [`gas/Code.gs`](gas/Code.gs) を貼り付け
3. プロジェクトの設定 → **スクリプト プロパティ** に `SECRET = 任意の合言葉` を追加（推奨）
4. **デプロイ → 新しいデプロイ → ウェブアプリ**
   - 実行するユーザー：自分 ／ アクセスできるユーザー：**全員**
5. 発行された `https://script.google.com/macros/s/XXXX/exec` のURLを控える

### Node側（Render）の設定
Renderのサービスの **Environment** に以下を設定（`render.yaml`にも枠を用意済み）：

| 変数 | 値 |
|------|----|
| `GAS_URL` | 上で控えた `…/exec` のURL |
| `GAS_SECRET` | GASの`SECRET`と同じ合言葉 |
| `TZ_DISPLAY` | `Asia/Tokyo`（既定。月境界のタイムゾーン） |

> ブラウザは直接GASを叩かず、**Nodeサーバーがサーバー間通信でGASを読み書き**します（CORS不要・SECRETで保護）。
> スプレッドシートは `month / duration / name / location / timestamp` 列で、人間が見て・編集して管理できます。

## ローカル / 自分のPCで動かす

### 必要なもの
- Node.js 18 以降
- ディスプレイ用の端末（PC・タブレット・スマホいずれか）
- 挑戦者用のスマホ（QRスキャン用）

### セットアップ & 起動

```bash
npm install
npm start
```

**ディスプレイ端末のブラウザで、スマホからも届くアドレスを開く**と早撃ち台になります。
2台で遊ぶときは `localhost` ではなく、PCのLAN IP（例 `http://192.168.0.2:3000/`）で開いてください。
QRにはそのアクセス元のホストが自動で使われるので、スマホが同じWi-Fiなら届きます。

## 遊び方

1. ディスプレイURLをPC等で開く（全画面推奨）
2. 画面を**クリック / スペースキー**で「抜き始め」
3. 数秒の溜めのあと、画面にQRが出現（＝抜けェ！）
4. 挑戦者がスマホのカメラ/QRリーダーで素早くスキャン → リンクを開く
5. 開いた瞬間にタイム確定。スマホに結果が表示され、ランキング入りなら名前を登録
6. もう一度クリック/スペースで次のラウンドへ

### 対戦・妨害（freeze）

元QRGunman同様、待機中の相手を妨害する **freeze** を搭載。
`/freeze` にアクセスすると、待機中なら抜きのタイミングが5〜8秒に延長されます（画面が凍りつく演出）。

### QRに使われる公開URLの決まり方

QRに埋め込むURLは「**ディスプレイを開いたときのアクセス元ホスト**」から自動で決まります。

- **同一Wi-Fi（LAN）**：PCのLAN IP（例 `http://192.168.0.2:3000/`）でディスプレイを開けば、QRもそのIPになり、同じWi-Fiのスマホで届く。
- **クラウド（Render等）**：公開ドメインが自動採用されるので設定不要。
- **明示指定したいとき**：環境変数 `PUBLIC_URL` を設定すると最優先で使われる（cloudflaredトンネル等）。

```bash
# 例: cloudflaredで公開し、そのURLをQRに固定したい場合
cloudflared tunnel --url http://localhost:3000
PUBLIC_URL=https://xxxx.trycloudflare.com npm start
```
> Windows PowerShell: `$env:PUBLIC_URL="https://xxxx.trycloudflare.com"; npm start`

## 設定（環境変数）

| 変数 | 既定 | 説明 |
|------|------|------|
| `PORT` | `3000` | 待ち受けポート |
| `PUBLIC_URL` | 自動 | QRに埋め込む公開ベースURLを明示指定（最優先） |
| `RENDER_EXTERNAL_URL` | Renderが自動設定 | Render公開URL（自分で設定不要） |
| `GAS_URL` | なし | ランキング保存先のGAS Webアプリ `…/exec`。未設定なら`records.json` |
| `GAS_SECRET` | なし | GAS書き込み保護の合言葉（GASの`SECRET`と一致させる） |
| `TZ_DISPLAY` | `Asia/Tokyo` | 月境界・日時表示のタイムゾーン |

ランキング保持数は `server.js` の `RECORD_MAX`（既定10）で変更できます。
ランキングは**月別（YYYY-MM, JST）に自動リセット**され、`GAS_URL`設定時はスプレッドシートに永続化されます。

## ファイル構成

```
QRevolver/
├── server.js              ゲーム本体（状態機械・計測・SSE・QR生成・月別ランキング）
├── public/
│   ├── display.html       ディスプレイ画面（M5Stackの置き換え）
│   ├── play.html          スマホ側の結果・登録ページ
│   └── assets/            元QRGunmanの画像・効果音を流用
├── gas/
│   └── Code.gs            ランキング保存用GAS Webアプリ（スプレッドシートDB）
├── records.json           ローカル保存フォールバック（GAS未設定時）
├── render.yaml            Renderデプロイ設定（Blueprint）
└── package.json
```

## 元プロジェクトからの主な改良点

- **実機不要**：M5Stackなしで、ありもののPC/スマホが早撃ち台に
- **動的QR**：ラウンド毎に使い捨てURL → 先読みチート防止
- **大画面対応**：背景アートとQRを画面いっぱいにレスポンシブ表示
- **同一時計計測の維持**：サーバー集約で端末間の時計ズレに影響されない正確なタイム
- 効果音・西部劇テイスト・ランキング・妨害(freeze)・AIガンマンの一言を継承
